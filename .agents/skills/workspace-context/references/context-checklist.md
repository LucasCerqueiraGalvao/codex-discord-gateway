# Context Checklist

Usar esta checklist para garantir que o contexto entregue seja acionavel.

## Campos minimos

- nome ou caminho do workspace
- objetivo do projeto
- stack principal
- alvo ativo mais provavel
- 2 a 5 caminhos-chave
- comandos uteis que tenham evidencia no repo
- configuracoes ou envs relevantes
- onde olhar logs ou traces
- superficies provaveis de edicao
- proxima leitura sugerida

## Quando aprofundar

Aprofundar somente se a tarefa seguinte depender disso, por exemplo:

- diferenciar apps em monorepo
- descobrir pipeline de deploy
- localizar origem de logs
- separar backend de frontend
- entender contrato de integracao ou fila

## Quando parar

Parar quando ja existir contexto suficiente para a proxima acao com baixa ambiguidade.

Se a resposta estiver virando inventario de repositorio, voce passou do ponto.
